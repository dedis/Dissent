Detailed Description

template<typename T>
class Dissent::Anonymity::RoundStateMachine< T >

Used as an internal mechanism to handle state logic within a Round.

Rounds HAVE a RoundStateMachine and never ARE a RoundStateMachine. I looked at QStateMachine and while it was quite generic, it overly focused on generics and didn't allow for inter-round callbacks or knowledge about cycles. It seemed better to encapsulate that behavior and construct this instead.

Make a RoundStateMachineImpl for inheritance purposes, so classes can properly implement the necessary behaviors for RoundStateMachine.
